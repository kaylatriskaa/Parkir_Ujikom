================================================================================
1. GAMBARAN UMUM PROJEK
================================================================================

Nama Aplikasi : Parkiest (Parking Management System)
Tipe          : Aplikasi Web
Framework     : Laravel 12 (PHP)
Database      : MySQL (nama database: ukk_parkir)
Frontend      : Blade Template + Tailwind CSS + Alpine.js
Grafik        : Chart.js (diagram batang pendapatan)
Font          : Plus Jakarta Sans (Google Fonts)
Icon          : Font Awesome 6

Parkiest adalah sistem manajemen parkir berbasis web yang dibangun
menggunakan framework Laravel. Aplikasi ini mengelola seluruh siklus
operasional parkir: mulai dari kendaraan masuk, pencatatan otomatis,
monitoring slot real-time, sampai kendaraan keluar dan pembayaran.

Aplikasi ini punya 3 jenis akun (role) dengan hak akses berbeda:
  1. Admin   → mengelola data master (user, tarif, area parkir)
  2. Petugas → operasional harian (input masuk/keluar, cetak struk)
  3. Owner   → pemantauan pendapatan, grafik, dan export laporan PDF

Setiap role punya dashboard sendiri-sendiri yang hanya bisa diakses
oleh role tersebut (dilindungi oleh middleware CheckRole).

2. FITUR LENGKAP PER ROLE
================================================================================

ADMIN (Master Data Management):

    - Stat Card: Total Petugas (jumlah akun petugas yang terdaftar)

    - Tabel semua user (kecuali admin yang login)

    - Tombol Edit → modal popup → ubah data user
    - Tombol Hapus → konfirmasi → hapus user dari database
    - Toggle Aktif/Nonaktif → user yang dinonaktifkan tidak bisa login

  Tab "Tarif & Area":
    - Daftar Tarif:
      → Tabel jenis kendaraan + harga per jam
      → Tombol Tambah Tarif → modal popup → isi jenis + harga
      → Tombol Edit → ubah harga per jam
    - Daftar Area Parkir:
      → Card per area: nama, kapasitas, terisi, progress bar
      → Tombol Tambah Area → modal popup → isi nama + kapasitas
      → Tombol Edit → ubah nama/kapasitas
      → Toggle Enable/Disable → area nonaktif kapasitasnya jadi 0

  Tab "Log Aktivitas":
    - Tabel 20 transaksi terbaru (plat, area, waktu, status, biaya)

PETUGAS (Operator Lapangan):
  - Jam digital real-time (update setiap detik)
  - Tanggal hari ini dalam bahasa Indonesia

  Tab "Masuk":
    - Dropdown pilihan jenis kendaraan (dari tabel tarif)
    - Dropdown pilihan area parkir (hanya yang masih ada slot)
    - Input plat nomor (otomatis huruf kapital, hapus spasi)
    - Tombol "Cetak Karcis" → proses masuk + auto-print struk
    - Validasi: cek dobel parkir + cek slot penuh

  Tab "Keluar":
    - Input plat nomor kendaraan yang mau keluar
    - Otomatis tampilkan: info kendaraan, durasi parkir, biaya
    - Modal konfirmasi pembayaran → proses bayar → auto-print struk LUNAS

  Tab "Parkir":
    - Daftar kendaraan yang SEDANG PARKIR (live, real-time)
    - Info per kendaraan: plat, jenis, waktu masuk, durasi, estimasi biaya
    - Estimasi biaya dihitung otomatis setiap detik (JavaScript)

  Tab "Aktivitas":
    - Riwayat transaksi berdasarkan tanggal (bisa difilter)
    - Tabel: plat, jenis, area, status, waktu masuk, waktu keluar, biaya

  Tab "Area":
    - Denah visual slot parkir per area
    - Dropdown pilih area → tampil grid slot (hijau = kosong, merah = terisi)
    - Progress bar kapasitas terpakai

  Struk:
    - Karcis Masuk → struk thermal 80mm, otomatis print
    - Struk Keluar → struk pembayaran dengan stempel LUNAS

OWNER (Pemilik Bisnis):
  - Stat Card: Total Pendapatan (dalam periode yang dipilih)
  - Stat Card: Total Kendaraan (dalam periode)
  - Stat Card: Total Petugas terdaftar
  - Stat Card: Total Area parkir

  - Filter Tanggal: pilih periode awal-akhir → klik "Sinkronkan Data"
  - Grafik Batang: pendapatan 7 hari terakhir (Chart.js)
  - Tabel Rincian Pendapatan: 10 transaksi terakhir yang sudah bayar
  - Tabel Log Aktivitas: 12 transaksi terbaru

    - Tombol "Export PDF" → buka popup window
    - Halaman laporan rapi dengan logo, tabel lengkap, ringkasan
    - Klik Print (Ctrl+P) → pilih "Save as PDF"

3. DATABASE (8 TABEL)
================================================================================

Nama database: ukk_parkir
File schema: schema.sql (di root folder projek)

TABEL 1: tb_user (Data Akun Pengguna)
  - id_user (INT, PK, Auto Increment) → ID unik setiap user
  - nama_lengkap (VARCHAR 100) → Nama lengkap user
  - username (VARCHAR 50, UNIQUE) → Username untuk login
  - password (VARCHAR 255) → Password sudah di-hash MD5
  - role (ENUM: admin, petugas, owner) → Jenis akun
  - status_aktif (TINYINT, Default: 1) → 1=aktif, 0=dinonaktifkan

TABEL 2: tb_tarif (Harga Parkir)
  - id_tarif (INT, PK, Auto Increment) → ID unik tarif
  - jenis_kendaraan (VARCHAR 50) → Motor, Mobil, dll
  - tarif_per_jam (DECIMAL 10,2) → Harga per jam

TABEL 3: tb_area_parkir (Area/Zona Parkir)
  - id_area (INT, PK, Auto Increment) → ID unik area
  - nama_area (VARCHAR 100) → Nama area (misal: Gedung A)
  - kapasitas (INT) → Total slot tersedia (0 = nonaktif)
  - terisi (INT) → Jumlah slot yang sedang dipakai

TABEL 4: tb_kendaraan (Data Kendaraan)
  - id_kendaraan (INT, PK, Auto Increment) → ID unik kendaraan
  - plat_nomor (VARCHAR 20) → Plat nomor (misal: B1234ABC)
  - jenis_kendaraan (VARCHAR 50) → Motor/Mobil
  - warna (VARCHAR 50) → Warna kendaraan
  - pemilik (VARCHAR 100) → Nama pemilik
  - id_user (INT, FK → tb_user) → Petugas yang pertama kali input

TABEL 5: tb_transaksi (Transaksi Parkir)
  - id_parkir (INT, PK, Auto Increment) → ID unik transaksi
  - id_kendaraan (INT, FK → tb_kendaraan)
  - id_tarif (INT, FK → tb_tarif)
  - id_area (INT, FK → tb_area_parkir)
  - id_user (INT, FK → tb_user) → Petugas yang menangani
  - waktu_masuk (DATETIME)
  - waktu_keluar (DATETIME, Nullable) → kosong kalau belum keluar
  - durasi_jam (INT, Nullable) → dihitung saat keluar
  - biaya_total (DECIMAL, Nullable) → dihitung saat keluar
  - status (ENUM: masuk, keluar) → status transaksi

TABEL 6: tb_log_aktivitas (Log Aktivitas User)
  - id_log (INT, PK) → ID log
  - id_user (INT, FK → tb_user) → User yang melakukan aktivitas
  - aktivitas (TEXT) → Deskripsi aktivitas
  - waktu (DATETIME) → Waktu aktivitas

TABEL 7: tb_log (Log Tambahan)
  - id_log (INT, PK) → ID log
  - id_user (INT, FK → tb_user)
  - aksi (VARCHAR 255) → Jenis aksi
  - detail (TEXT) → Detail aksi
  - created_at (DATETIME)

TABEL 8: sessions (Session Laravel)
  - id (VARCHAR, PK) → Session ID
  - user_id, ip_address, user_agent, payload, last_activity

RELASI ANTAR TABEL:
  tb_transaksi → id_kendaraan → tb_kendaraan
  tb_transaksi → id_tarif → tb_tarif
  tb_transaksi → id_area → tb_area_parkir
  tb_transaksi → id_user → tb_user
  tb_kendaraan → id_user → tb_user

4. ARSITEKTUR MVC (MODEL - VIEW - CONTROLLER)
================================================================================

MVC adalah pola desain yang memisahkan aplikasi jadi 3 bagian:

MODEL (app/Models/) → Penghubung ke Database
  Setiap tabel di database punya 1 model.
  Model menentukan:
  - Tabel mana yang dihubungkan ($table)
  - Kolom mana yang boleh diisi ($fillable)
  - Relasi antar tabel (belongsTo, hasMany)
  
  Contoh di projek:
    User.php      → tb_user
    Tarif.php     → tb_tarif
    Area.php      → tb_area_parkir
    Kendaraan.php → tb_kendaraan
    Transaksi.php → tb_transaksi

VIEW (resources/views/) → Tampilan HTML
  File-file Blade (.blade.php) yang menghasilkan HTML.
  Blade Template punya fitur khusus:
    {{ $variabel }}     → cetak variabel
    @foreach ... @endforeach → looping
    @if ... @endif      → kondisi
    @extends, @section  → template inheritance

CONTROLLER (app/Http/Controllers/) → Logika/Otak Aplikasi
  Setiap role punya controller sendiri:
    AdminController.php   → logika halaman admin
    PetugasController.php → logika halaman petugas
    OwnerController.php   → logika halaman owner

ROUTES (routes/web.php) → Peta URL
  Menentukan URL mana ditangani oleh fungsi Controller mana.

ALUR KERJA:
  Browser → URL → Routes → Controller → Model → Database
  Database → Model → Controller → View → Browser

5. SISTEM AUTENTIKASI (LOGIN/LOGOUT)
================================================================================

File-file terkait:
  - resources/views/auth/login.blade.php → Halaman login (form)
  - app/Http/Requests/Auth/LoginRequest.php → Proses autentikasi MD5
  - app/Http/Controllers/Auth/AuthenticatedSessionController.php → Setelah login

ALUR LOGIN:
  1. User buka /login → tampil halaman login (form username + password)
  2. User isi form → klik LOGIN → data dikirim ke server (POST)
  3. LoginRequest menerima data → validasi (wajib diisi)
  4. Cari user di database berdasarkan username
  5. Hash password yang diketik pakai MD5
  6. Bandingkan hash dengan password di database
  7. Kalau COCOK → login berhasil → cek status_aktif
  8. Kalau status_aktif = 0 → paksa logout + pesan error
  9. Kalau status_aktif = 1 → redirect ke dashboard sesuai role

KEAMANAN:
  - Password dienkripsi MD5 (tidak disimpan dalam bentuk asli)
  - Rate limiting: maksimal 5x gagal login → blokir sementara
  - Session regeneration: mencegah session fixation attack
  - CSRF Token: mencegah pemalsuan form

================================================================================
6. MIDDLEWARE KEAMANAN (RBAC)
================================================================================

File: app/Http/Middleware/CheckRole.php

RBAC = Role-Based Access Control (kontrol akses berdasarkan role)

Middleware ini berjalan SEBELUM halaman dibuka. Seperti satpam:
  1. Cek: sudah login? → kalau belum, tendang ke /login
  2. Cek: role-nya boleh akses? → kalau boleh, lanjut buka halaman
  3. Kalau tidak boleh → redirect ke dashboard sendiri + pesan error

Contoh:
  Petugas coba buka /admin/dashboard
  → CheckRole cek: role 'petugas' ada di daftar ['admin']?
  → TIDAK → redirect ke /petugas/dashboard

7. ALUR KERJA PARKIR LENGKAP
================================================================================

KENDARAAN MASUK:
  1. Petugas pilih jenis kendaraan (Motor/Mobil) dari dropdown
  2. Petugas pilih area parkir dari dropdown (hanya yang masih ada slot)
  3. Petugas input plat nomor kendaraan
  4. Klik tombol "Cetak Karcis"
  5. Sistem VALIDASI:
     a. Cek: plat nomor sudah diisi? Jenis sudah dipilih? Area sudah dipilih?
     b. Cek: kendaraan dengan plat ini masih parkir di dalam?
        → Kalau iya → TOLAK (tidak bisa masuk 2x)
     c. Cek: area yang dipilih masih ada slot kosong?
        → Kalau penuh → TOLAK
  6. Sistem SIMPAN:
     a. Cari/buat data kendaraan di database (firstOrCreate)
     b. Buat transaksi baru (status: 'masuk', waktu_masuk: sekarang)
     c. Tambah slot terisi di area +1 (increment)
  7. Auto-print KARCIS MASUK (struk thermal 80mm)

KENDARAAN KELUAR:
  1. Petugas input plat nomor kendaraan yang mau keluar
  2. JavaScript otomatis cari data: tampilkan jenis, waktu masuk, durasi, biaya
  3. Klik tombol "Proses Pembayaran"
  4. Muncul modal konfirmasi: total biaya yang harus dibayar
  5. Klik "Bayar" → sistem PROSES:
     a. Hitung durasi parkir (waktu masuk → waktu sekarang, minimal 1 jam)
     b. Hitung biaya (durasi × tarif per jam, dibulatkan ke atas)
     c. Update transaksi: isi waktu_keluar, durasi, biaya, status = 'keluar'
     d. Kurangi slot terisi di area -1 (decrement)
  6. Auto-print STRUK KELUAR (struk pembayaran dengan stempel LUNAS)

1. GAMBARAN UMUM PROJEK
================================================================================

Aplikasi ini adalah Sistem Manajemen Parkir berbasis web yang dibangun
menggunakan framework Laravel 12. Aplikasi ini memiliki 3 role pengguna:

- ADMIN    : Mengelola user (CRUD), mengatur tarif, dan monitoring area parkir
- PETUGAS  : Menginput kendaraan masuk/keluar, mencetak karcis, melihat riwayat
- OWNER    : Melihat laporan pendapatan, grafik, dan monitoring traffic

Fitur Utama:
- Login multi-role (Admin, Petugas, Owner)
- Manajemen kendaraan masuk & keluar real-time
- Cetak karcis/struk otomatis
- Perhitungan biaya parkir otomatis berdasarkan durasi
- Dashboard monitoring area parkir dengan visualisasi slot
- Laporan pendapatan dengan grafik (Chart.js)
- Filter data berdasarkan tanggal

2. STRUKTUR DATABASE (SCHEMA)
================================================================================

Nama Database: ukk_parkir
File Schema  : schema.sql

Tabel-tabel yang digunakan:

--- 2.1 tb_user ---
Menyimpan data pengguna sistem.
Kolom:
  - id_user       : Primary Key, auto increment
  - nama_lengkap  : Nama lengkap pengguna (VARCHAR 100)
  - username      : Username untuk login, harus unik (VARCHAR 50)
  - password      : Password disimpan dalam format MD5 (VARCHAR 255)
  - role          : Peran pengguna, pilihan: 'admin', 'petugas', 'owner'
  - status_aktif  : Status akun, 1 = Aktif, 0 = Nonaktif (TINYINT)

Penjelasan:
  Password menggunakan enkripsi MD5. Contoh: password "admin123" akan
  disimpan sebagai "0192023a7bbd73250516f069df18b500".
  Field role menggunakan tipe ENUM yang membatasi isi hanya 3 pilihan.
  Field status_aktif memungkinkan admin menonaktifkan akun tanpa menghapusnya.

--- 2.2 tb_tarif ---
Menyimpan tarif parkir per jenis kendaraan.
Kolom:
  - id_tarif        : Primary Key
  - jenis_kendaraan : Jenis kendaraan (misal: motor, mobil)
  - tarif_per_jam   : Harga parkir per jam dalam Rupiah

Penjelasan:
  Tabel ini sederhana, hanya menyimpan jenis dan harga. Saat kendaraan
  masuk, sistem akan mengambil tarif dari sini untuk menghitung biaya.

--- 2.3 tb_area_parkir ---
Menyimpan data area/zona parkir.
Kolom:
  - id_area    : Primary Key
  - nama_area  : Nama area (misal: Lantai 1, Blok A)
  - kapasitas  : Total slot yang tersedia
  - terisi     : Jumlah slot yang sedang terpakai

Penjelasan:
  Kolom "terisi" akan bertambah 1 saat kendaraan masuk (increment),
  dan berkurang 1 saat kendaraan keluar (decrement).
  Slot kosong dihitung: kapasitas - terisi.

--- 2.4 tb_kendaraan ---
Menyimpan identitas kendaraan yang pernah parkir.
Kolom:
  - id_kendaraan    : Primary Key
  - plat_nomor      : Nomor plat kendaraan
  - jenis_kendaraan : Motor atau mobil
  - warna           : Warna kendaraan
  - pemilik         : Nama pemilik kendaraan
  - id_user         : Foreign Key ke tb_user (petugas yang mendaftarkan)

Penjelasan:
  Tabel ini memisahkan data kendaraan dari transaksi. Satu kendaraan
  bisa memiliki banyak transaksi parkir (relasi one-to-many).

--- 2.5 tb_transaksi ---
Tabel utama transaksi parkir.
Kolom:
  - id_parkir    : Primary Key
  - id_kendaraan : Foreign Key ke tb_kendaraan
  - id_tarif     : Foreign Key ke tb_tarif
  - id_area      : Foreign Key ke tb_area_parkir
  - id_user      : Foreign Key ke tb_user (petugas yang melayani)
  - waktu_masuk  : Waktu kendaraan masuk (DATETIME)
  - waktu_keluar : Waktu kendaraan keluar (DATETIME, nullable)
  - durasi_jam   : Lama parkir dalam jam
  - biaya_total  : Total biaya yang harus dibayar
  - status       : 'masuk' atau 'keluar'

Penjelasan:
  Saat kendaraan masuk: status='masuk', waktu_keluar=NULL, biaya_total=0
  Saat kendaraan keluar: status='keluar', waktu_keluar diisi, durasi &
  biaya dihitung otomatis.

--- 2.6 tb_log_aktivitas & tb_log ---
Tabel pencatatan aktivitas sistem (untuk audit trail).

RELASI ANTAR TABEL:
  tb_transaksi.id_kendaraan -> tb_kendaraan.id_kendaraan
  tb_transaksi.id_tarif     -> tb_tarif.id_tarif
  tb_transaksi.id_area      -> tb_area_parkir.id_area
  tb_transaksi.id_user      -> tb_user.id_user
  tb_kendaraan.id_user      -> tb_user.id_user

3. ARSITEKTUR APLIKASI (MVC)
================================================================================

Laravel menggunakan pola MVC (Model-View-Controller):

MODEL (app/Models/)
  = Representasi tabel database di dalam kode PHP
  = Mengatur relasi antar tabel
  = Contoh: Model User.php merepresentasikan tabel tb_user

VIEW (resources/views/)
  = Tampilan yang dilihat user di browser
  = Menggunakan Blade Template Engine (file .blade.php)
  = Contoh: login.blade.php adalah halaman login

CONTROLLER (app/Http/Controllers/)
  = Otak logika aplikasi
  = Menerima request dari user, memproses data, mengembalikan response
  = Contoh: PetugasController menangani logika parkir masuk/keluar

ALUR KERJA MVC:
  1. User membuka URL (misal: /petugas/dashboard)
  2. Routes (routes/web.php) mencocokkan URL ke Controller
  3. Controller mengambil data dari Model (database)
  4. Controller mengirim data ke View (blade template)
  5. View menampilkan halaman HTML ke browser user

4. SISTEM AUTENTIKASI (LOGIN)
================================================================================

FILE: app/Http/Requests/Auth/LoginRequest.php

Cara kerja login:
  1. User memasukkan username dan password di form login
  2. Sistem mencari user di tabel tb_user berdasarkan username
  3. Password yang diketik user di-hash dengan MD5
  4. Hash MD5 dibandingkan dengan password di database
  5. Jika cocok, user di-login-kan dan diarahkan ke dashboard sesuai role
  6. Jika tidak cocok, muncul pesan error

Kode penting (LoginRequest.php):
----------------------------------------------------------------------
  public function authenticate(): void
  {
      $user = User::where('username', $this->username)->first();

      if (!$user || md5($this->password) !== $user->password) {
          throw ValidationException::withMessages([
              'username' => trans('auth.failed'),
          ]);
      }

      Auth::login($user, $this->boolean('remember'));
  }
----------------------------------------------------------------------
Penjelasan baris per baris:
  - Baris 1: Cari user di database berdasarkan username yang diketik
  - Baris 3: Cek apakah user ditemukan DAN password MD5-nya cocok
             md5('admin123') = '0192023a7bbd73250516f069df18b500'
  - Baris 4-6: Jika gagal, lempar error "Username atau password salah"
  - Baris 8: Jika berhasil, login-kan user ke sistem Laravel

FILE: app/Http/Controllers/Auth/AuthenticatedSessionController.php
Setelah login berhasil, controller ini mengecek status_aktif:
----------------------------------------------------------------------
  if ($request->user()->status_aktif == 0) {
      Auth::guard('web')->logout();
      return redirect()->route('login')->withErrors([
          'username' => 'Akun Anda telah dinonaktifkan oleh Admin.',
      ]);
  }

  $role = $request->user()->role;
  return match ($role) {
      'admin'   => redirect()->route('admin.dashboard'),
      'petugas' => redirect()->route('petugas.dashboard'),
      'owner'   => redirect()->route('owner.dashboard'),
  };
----------------------------------------------------------------------
Penjelasan:
  - Cek apakah akun masih aktif (status_aktif = 1)
  - Jika nonaktif, logout paksa dan tampilkan pesan error
  - Jika aktif, arahkan ke dashboard sesuai role (admin/petugas/owner)

FILE VIEW: resources/views/auth/login.blade.php
  - Form login dengan input username dan password
  - Menggunakan animasi mobil dan palang parkir (CSS animation)
  - Tampilan split-screen: kiri animasi, kanan form login

5. MIDDLEWARE & KEAMANAN (RBAC)
================================================================================

RBAC = Role-Based Access Control (Kontrol Akses Berdasarkan Peran)

FILE: app/Http/Middleware/CheckRole.php
----------------------------------------------------------------------
  public function handle(Request $request, Closure $next, ...$roles)
  {
      if (!Auth::check()) {
          return redirect('login');
      }

      if (in_array(Auth::user()->role, $roles)) {
          return $next($request);
      }

      return redirect()->route(Auth::user()->role . '.dashboard')
                       ->with('error', 'Anda tidak memiliki akses.');
  }
----------------------------------------------------------------------
Penjelasan:
  - Parameter ...$roles menerima daftar role yang diizinkan
  - Langkah 1: Cek apakah user sudah login, jika belum redirect ke login
  - Langkah 2: Cek apakah role user ada di daftar yang diizinkan
  - Langkah 3: Jika tidak punya akses, redirect ke dashboard miliknya

Contoh penggunaan di routes/web.php:
  Route::middleware(['auth', 'role:admin']) -> hanya admin yang bisa akses
  Route::middleware(['auth', 'role:petugas']) -> hanya petugas yang bisa akses

FILE: bootstrap/app.php
  Mendaftarkan middleware 'role' agar bisa dipakai di routes:
  $middleware->alias(['role' => CheckRole::class]);

6. MODEL - PENJELASAN LENGKAP
================================================================================

--- 6.1 User.php (app/Models/User.php) ---
----------------------------------------------------------------------
  protected $table = 'tb_user';           // Nama tabel di database
  protected $primaryKey = 'id_user';      // Kolom primary key
  public $timestamps = false;             // Tidak pakai created_at/updated_at

  protected $fillable = [                 // Kolom yang boleh diisi massal
      'nama_lengkap', 'username', 'password', 'role', 'status_aktif',
  ];
----------------------------------------------------------------------
Penjelasan:
  - $table: Memberitahu Laravel nama tabel yang dipakai (default: 'users')
  - $primaryKey: Memberitahu kolom ID (default: 'id')
  - $timestamps: Laravel biasanya otomatis mengisi created_at & updated_at,
    kita matikan karena tabel kita tidak punya kolom tersebut
  - $fillable: Daftar kolom yang boleh diisi dengan mass assignment
    (keamanan agar kolom sensitif tidak bisa dimanipulasi)

--- 6.2 Kendaraan.php (app/Models/Kendaraan.php) ---
----------------------------------------------------------------------
  protected $table = 'tb_kendaraan';
  protected $primaryKey = 'id_kendaraan';

  public function user() {
      return $this->belongsTo(User::class, 'id_user', 'id_user');
  }
  public function transaksis() {
      return $this->hasMany(Transaksi::class, 'id_kendaraan', 'id_kendaraan');
  }
----------------------------------------------------------------------
Penjelasan relasi:
  - belongsTo(User) = "Kendaraan ini DIMILIKI OLEH satu User (petugas)"
    Artinya: satu kendaraan didaftarkan oleh satu petugas
  - hasMany(Transaksi) = "Kendaraan ini MEMILIKI BANYAK Transaksi"
    Artinya: satu kendaraan bisa parkir berkali-kali

--- 6.3 Transaksi.php (app/Models/Transaksi.php) ---
----------------------------------------------------------------------
  protected $table = 'tb_transaksi';
  protected $primaryKey = 'id_parkir';

  public function kendaraan() {
      return $this->belongsTo(Kendaraan::class, 'id_kendaraan', 'id_kendaraan');
  }
  public function tarif() {
      return $this->belongsTo(Tarif::class, 'id_tarif', 'id_tarif');
  }
  public function area() {
      return $this->belongsTo(Area::class, 'id_area', 'id_area');
  }
  public function user() {
      return $this->belongsTo(User::class, 'id_user', 'id_user');
  }
----------------------------------------------------------------------
Penjelasan:
  Model Transaksi punya 4 relasi belongsTo karena satu transaksi
  terhubung ke: 1 kendaraan, 1 tarif, 1 area, dan 1 user (petugas).

--- 6.4 Tarif.php & Area.php ---
  Sama polanya: mendefinisikan $table, $primaryKey, $fillable.
  Area memiliki kolom 'terisi' yang di-increment/decrement saat
  kendaraan masuk/keluar.

7. CONTROLLER - PENJELASAN LENGKAP
================================================================================

--- 7.1 AdminController.php ---
FILE: app/Http/Controllers/Admin/AdminController.php

Method index() - Menampilkan dashboard admin:
  - Mengambil semua user (petugas & owner)
  - Mengambil semua tarif dan area
  - Mengirim data ke view dashboards/admin.blade.php

Method store() - Menambah user baru:
  1. Validasi input (nama, username unik, password min 6 karakter)
  2. Password di-hash dengan md5() sebelum disimpan
  3. Simpan user baru ke database
  4. Redirect balik dengan pesan sukses

Method update() - Edit data user:
  1. Cari user berdasarkan ID
  2. Validasi input (username harus unik, kecuali milik user ini sendiri)
  3. Update data, password hanya diubah jika diisi
  4. Simpan perubahan

Method toggleStatus() - Aktifkan/Nonaktifkan akun:
  - Membalik nilai status_aktif (1 jadi 0, 0 jadi 1)
  - User yang dinonaktifkan tidak bisa login

Method updateTarif() - Mengubah tarif parkir:
  - Validasi harga baru
  - Update kolom tarif_per_jam

--- 7.2 PetugasController.php ---
FILE: app/Http/Controllers/Petugas/PetugasController.php

Method index() - Dashboard petugas:
  - Mengambil semua tarif untuk dropdown
  - Mengambil area yang masih ada slot kosong
  - Mengambil daftar kendaraan yang sedang parkir (status='masuk')
  - Mengambil riwayat transaksi berdasarkan tanggal filter

Method kendaraanMasuk() - Proses kendaraan masuk:
----------------------------------------------------------------------
  1. Validasi input (plat nomor, tarif, area)
  2. Cek apakah plat nomor sudah ada yang parkir (cegah duplikat)
  3. Cek apakah area masih ada slot kosong
  4. Simpan/cari data kendaraan di tb_kendaraan (firstOrCreate)
  5. Buat transaksi baru dengan status='masuk'
  6. Tambah kolom 'terisi' di area (+1)
  7. Redirect dengan flash session untuk cetak karcis
----------------------------------------------------------------------
Kode penting:
  $kendaraan = Kendaraan::firstOrCreate(
      ['plat_nomor' => $platNomor],       // Cari berdasarkan ini
      ['jenis_kendaraan' => ..., ...]     // Buat baru jika tidak ketemu
  );
Penjelasan firstOrCreate:
  Kalau kendaraan dengan plat itu sudah pernah parkir, pakai data lama.
  Kalau belum pernah, buat data kendaraan baru.

Method kendaraanKeluar() - Proses kendaraan keluar:
----------------------------------------------------------------------
  1. Cari transaksi aktif berdasarkan plat nomor
  2. Hitung durasi parkir (waktu keluar - waktu masuk)
  3. Update transaksi: waktu_keluar, durasi_jam, biaya_total, status='keluar'
  4. Kurangi kolom 'terisi' di area (-1)
  5. Redirect dengan flash session 'terbayar' untuk menampilkan struk LUNAS
----------------------------------------------------------------------

Method cetakKarcis() - Cetak struk masuk:
  - Mengambil data transaksi beserta relasi (kendaraan, area, tarif)
  - Menampilkan halaman cetak yang otomatis membuka dialog print

--- 7.3 OwnerController.php ---
FILE: app/Http/Controllers/Owner/OwnerController.php

Method index() - Dashboard owner:
  - Menerima filter tanggal dari request (start_date, end_date)

10. ALUR KERJA SISTEM (FLOW)
================================================================================

--- ALUR KENDARAAN MASUK ---
  1. Petugas buka Tab "Masuk" di dashboard
  2. Pilih jenis kendaraan (motor/mobil) dari dropdown
  3. Pilih area parkir yang tersedia
  4. Ketik plat nomor kendaraan
  5. Klik "CETAK KARCIS"
  6. Sistem membuat data kendaraan (jika belum ada)
  7. Sistem membuat transaksi baru (status: masuk)
  8. Slot area bertambah 1 (terisi + 1)
  9. Karcis otomatis dicetak di tab baru

--- ALUR KENDARAAN KELUAR ---
  1. Petugas buka Tab "Keluar"
  2. Ketik plat nomor kendaraan
  3. Sistem mencari data di daftar kendaraan aktif (JavaScript)
  4. Otomatis menampilkan: waktu masuk, durasi, total biaya
  5. Klik "PROSES PEMBAYARAN"
  6. Modal muncul: total tagihan, input jumlah bayar, kembalian
  7. Klik "Bayar" jika uang cukup
  8. Sistem update transaksi (status: keluar, waktu_keluar, biaya)
  9. Slot area berkurang 1 (terisi - 1)
  10. Struk pembayaran tampil dengan stamp "LUNAS"

--- ALUR ADMIN KELOLA USER ---
  1. Admin login -> masuk dashboard Admin
  2. Tab "Manajemen User" tampil
  3. Klik "+ TAMBAH AKUN" -> isi form -> Simpan
  4. Klik icon Edit -> ubah data -> Simpan
  5. Klik icon Toggle -> aktifkan/nonaktifkan akun

--- ALUR OWNER LIHAT LAPORAN ---
  1. Owner login -> masuk dashboard Owner
  2. Pilih periode tanggal (start - end)
  3. Klik "Sinkronkan Data"
  4. Dashboard menampilkan: pendapatan, grafik, rincian
